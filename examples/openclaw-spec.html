Design Spec

A digital deputy for the asks Claude can't handle

2026-05-02ApprovedScope: ~/.claude/CLAUDE.md

When Claude is about to ask you to do something manually on this Mac, it should first check if OpenClaw can do it. This spec adds that check as one section in your user-level Claude file. Two tiers: auto-execute for safe chores, suggest-only for sensitive ones.

Why this exists

Claude Code can't do everything itself. It hits System Settings toggles, native dialogs, apps outside the browser — and asks you to handle them. Most of those asks are GUI chores, not real human work.

OpenClaw is already installed for that gap: local LLM agent, full GUI automation via Peekaboo, YOLO exec policy. The capability is here. This spec wires up the reflex.

How the handoff works

The instruction lives in one new file: ~/.claude/CLAUDE.md, loaded into every Claude Code conversation on this machine. No hooks, no scripts, no slash commands.

Each turn, Claude runs the decision tree in Figure 2. Most paths short-circuit: Claude does it itself, or only you can decide. The new path is when Claude needs help and OpenClaw can give it.

Two tiers, one boundary

The line between "Claude runs it" and "you run it" is what matters. It's drawn around reversibility, sensitivity, and scope. Examples on both sides anchor the call.

TierWhen it appliesWhat Claude doesExamples
Auto-execute Short, scoped, reversible · GUI / system Asks "Send to OpenClaw?" → runs via Bash → summarizes Open an app · click a button · toggle a System Setting · take a screenshot · fill a known form · send a Mac-app message
Suggest-only Sensitive · ambiguous · multi-app · > 3 min Prints the exact command; you decide Credentials · payments · irreversible state · login flows · anything unclear
Skip OpenClaw Out of OpenClaw's reach Asks the human directly, no OpenClaw mention Physical-world asks · judgment calls · "which design do you prefer?"

The instruction itself

The entire change is twenty-five lines of plain English in ~/.claude/CLAUDE.md:

## OpenClaw delegation

OpenClaw is at /opt/homebrew/bin/openclaw, gateway runs as LaunchAgent. Local LLM agent
with full GUI automation (Peekaboo: click, type, dialogs, screenshots, menus), YOLO exec
policy, embedded inference (~9s/turn). Treat as digital deputy.

When about to ask the human to manually do something on this computer that you cannot do
via your tools or MCP servers, first consider OpenClaw. It can plausibly handle: GUI
dialogs, app forms, System Settings, Mac apps (Mail, Messages, Calendar, Finder), browser
actions outside Chrome DevTools MCP, shell commands needing the user's environment.

**Auto-execute tier** — short, scoped, reversible GUI/system tasks:
ask "Want me to send this to OpenClaw?" → on yes, run via Bash:

    openclaw agent --local --message "<task>" --json --timeout 120

Parse JSON, summarize, continue. On failure, fall back to asking the human.

**Suggest-only tier** — sensitive (credentials, payments, irreversible, ambiguous,
multi-app, >3 min): print the exact command, do not auto-execute.

Skip the offer entirely for physical-world asks, judgment calls, or anything OpenClaw
clearly cannot help with.

When it breaks

OpenClaw can fail — wrong UI read, permission prompt, timeout, output Claude can't parse. In every case, Claude falls back to asking you.

Failure modeClaude's response
Non-zero exit or error JSONBrief error report, then ask the human directly
Result Claude can't interpretShow raw output, ask the human to confirm
Timeout (120 s)Treat as failure, fall back

The choices behind the design

Four decisions worth recording.

#ChoiceRejected alternativesReasoning
1Hybrid handoffPure-suggest; pure-execute; voice-onlyGUI chores are worth auto-executing; sensitive tasks still need a human in the loop.
2Prompt-only via CLAUDE.mdHook-enforced; slash command; bothThe judgment ("is this OpenClaw-fit?") is conversational, not regex-friendly. The failure mode of forgetting is mild and easy to correct.
3--local flag--gateway~9 s vs ~20 s for one-shot turns; gateway plumbing is wasted for single delegations.
4--timeout 120LongerBounds the auto-execute tier to "short scoped task". Anything longer belongs in suggest-only, where you control invocation.

What this is not

  • OpenClaw is a fallback, not a peer to Claude's tools.
  • Not every human-ask routes through OpenClaw.
  • No hooks, scripts, or slash commands. One paragraph in one file.
